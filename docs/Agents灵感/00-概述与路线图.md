# Mini-Todo Agent 调度系统 - 概述与路线图

## 1. 愿景

将 Mini-Todo 从"个人待办管理应用"升级为 **AI 驱动的代码任务调度中心**。

用户在 Todo 中描述需求，系统将任务派发给 AI Agent（Claude Code / Codex），Agent 执行后返回结果，用户审核确认。你就像一个项目经理，Mini-Todo 是你的任务面板，而 AI Agent 是你的开发团队。

## 2. 核心设计理念

| 理念 | 说明 |
|------|------|
| **任务即指令** | 每个 Todo 项可以是一条 Agent 指令，附带 Prompt、目标项目、执行策略 |
| **人机协作** | Agent 执行结果需人工审核确认，保证代码质量 |
| **多 Agent 协同** | 支持多种 AI Agent（Claude Code、Codex 等），按能力分工 |
| **渐进式复杂度** | 从单任务执行到工作流编排，逐步升级 |

## 4. 建议实施路线图

### 阶段一：MVP（最小可用产品）

**目标**：跑通 Agent 任务的完整闭环

- Agent 配置管理（支持 Claude Code CLI）
- 单任务派发与执行
- 实时执行日志展示
- 基础结果查看

**预期产出**：用户可以在 Mini-Todo 中创建一个 Agent 任务，指定项目路径和 Prompt，系统调用 Claude Code 执行并展示输出。

### 阶段二：核心体验

**目标**：让日常使用变得高效

- 多 Agent 支持（Claude Code + Codex）
- Prompt 模板系统
- Diff 预览与审核流程
- Git 分支自动管理
- 执行历史与成本统计

**预期产出**：用户可以用模板快速创建任务，Agent 在独立分支执行，完成后用户审核 Diff 并一键合并。

### 阶段三：智能调度

**目标**：自动化与智能化

- 任务依赖链
- 多项目管理
- 智能任务拆分
- 智能 Agent 分配
- 统计仪表盘

**预期产出**：用户输入一个大需求，系统自动拆分为子任务，分配给最合适的 Agent，按依赖关系自动调度。

### 阶段四：工作流引擎

**目标**：完整的自动化流水线

- 可视化工作流编辑器
- 条件分支与循环
- 人机协作节点
- 工作流模板库
- 定时/事件触发

**预期产出**：用户可以拖拽编排多步骤工作流，如"分析需求 -> 编写代码 -> 写测试 -> Code Review -> 创建 PR"。

## 5. 技术选型建议

| 模块 | 方案 | 说明 |
|------|------|------|
| Agent 通信 | Rust 子进程调用 CLI | 通过 `std::process::Command` 调用 Claude Code / Codex CLI |
| 任务队列 | SQLite + 定时轮询 | 复用现有 SQLite，新增 agent_tasks 表 |
| 实时日志 | Tauri 事件系统 | `emit` / `listen` 推送 Agent stdout 到前端 |
| Diff 渲染 | diff2html / Monaco Editor | 前端展示代码变更 |
| 工作流定义 | JSON Schema | 工作流存储为 JSON，SQLite 存执行实例 |
| Git 操作 | git2-rs (Rust) | 分支管理、Diff 生成、Commit 操作 |

## 6. 与现有功能的关系

Mini-Todo 的现有功能（待办管理、子任务、拖拽排序、通知提醒、窗口固定等）**完全保留**，Agent 调度系统作为**新增模块**叠加在现有能力之上：

- 普通 Todo 项：保持现有行为不变
- Agent Todo 项：新增类型，附带 Agent 执行能力
- 子任务可以是 Agent 自动拆分的结果
- 通知系统可扩展为 Agent 执行完成通知
- 四象限视图可以展示 Agent 任务的状态分布
