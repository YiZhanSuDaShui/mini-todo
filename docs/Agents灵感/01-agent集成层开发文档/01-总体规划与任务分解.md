# Agent 集成层 - 总体规划与任务分解

## 1. 开发目标

基于 [01-Agent集成层.md](../01-Agent集成层.md) 灵感文档，实现 Agent 集成层的 MVP 版本，使系统具备：

1. 配置和管理多个 AI Agent（Claude Code、Codex 等）
2. 通过统一的 CLI 适配器调用 Agent 执行代码任务
3. 实时流式展示 Agent 执行日志
4. 基于 Git Worktree 的安全隔离执行环境
5. API Key 的安全加密存储

## 3. 任务分解

### 阶段 A：数据层（预计 1-2 天）

| 编号 | 任务 | 描述 | 依赖 | 文件 |
|------|------|------|------|------|
| A1 | 数据库迁移 v8 | 创建 `agent_configs` 表 | 无 | `migrations.rs` |
| A2 | 数据模型定义 | Agent 相关的 Rust 结构体 | 无 | `db/models.rs` |
| A3 | 数据库操作函数 | Agent 配置的 CRUD 操作 | A1, A2 | `db/agent_db.rs`（新增） |
| A4 | 前端类型定义 | TypeScript 接口定义 | 无 | `types/agent.ts`（新增） |

### 阶段 B：核心服务层（预计 2-3 天）

| 编号 | 任务 | 描述 | 依赖 | 文件 |
|------|------|------|------|------|
| B1 | API Key 加密模块 | AES-256-GCM 加密/解密 | 无 | `services/agent/crypto.rs` |
| B2 | AgentRunner Trait | 统一的 CLI 适配器接口定义 | 无 | `services/agent/runner.rs` |
| B3 | Claude Code 适配器 | 实现 Claude Code 的 CLI 调用和 JSONL 解析 | B2 | `services/agent/claude_code.rs` |
| B4 | Codex 适配器 | 实现 Codex 的 CLI 调用和 JSONL 解析 | B2 | `services/agent/codex.rs` |
| B5 | Git Worktree 管理 | Worktree 创建/清理/Diff 收集 | 无 | `services/agent/worktree.rs` |
| B6 | 健康检查服务 | CLI 版本检测、可用性检查 | B2 | `services/agent/runner.rs` |

### 阶段 C：Tauri 命令层（预计 1 天）

| 编号 | 任务 | 描述 | 依赖 | 文件 |
|------|------|------|------|------|
| C1 | Agent 配置命令 | 增删改查 + 健康检查 Tauri 命令 | A3, B1 | `commands/agent_cmd.rs` |
| C2 | Agent 执行命令 | 执行、取消、状态查询命令 | B2-B5 | `commands/agent_cmd.rs` |
| C3 | 注册命令 | 在 lib.rs 中注册新命令 | C1, C2 | `lib.rs` |

### 阶段 D：前端实现（预计 2-3 天）

| 编号 | 任务 | 描述 | 依赖 | 文件 |
|------|------|------|------|------|
| D1 | Agent Store | Pinia 状态管理 | A4, C1 | `stores/agentStore.ts` |
| D2 | Agent 配置管理界面 | 设置页新增 Agent Tab | D1 | `components/AgentSettings.vue` |
| D3 | Agent 状态指示器 | 在线/离线/版本过低状态显示 | D1 | `components/AgentStatusBadge.vue` |
| D4 | 执行日志面板 | 终端风格的实时日志展示 | D1, C2 | `components/AgentLogPanel.vue` |

### 阶段 E：集成测试（预计 1 天）

| 编号 | 任务 | 描述 | 依赖 |
|------|------|------|------|
| E1 | 端到端流程测试 | 配置 Agent → 创建 Worktree → 执行 → 收集结果 | 全部 |
| E2 | 边界情况处理 | CLI 不存在、API Key 无效、超时、项目非 Git 仓库 | 全部 |

## 4. 新增依赖

### Rust (Cargo.toml)

| crate | 用途 | 版本 |
|-------|------|------|
| `aes-gcm` | AES-256-GCM 加密 | 最新 |
| `sha2` | SHA-256 哈希（密钥派生） | 最新 |
| `rand` | 加密随机数生成 | 最新 |
| `machine-uid` | 获取机器唯一标识（密钥派生） | 最新 |
| `async-trait` | 异步 Trait 支持 | 最新 |

> `tokio` 已存在，`serde_json` 已存在，`base64` 已存在，`git2-rs` 暂不引入（先用 `std::process::Command` 调用 git CLI）。

### 前端 (package.json)

| 包 | 用途 |
|-----|------|
| `xterm` + `@xterm/xterm` | （可选）终端风格日志渲染 |
| `ansi-to-html` | （可选）ANSI 颜色码转 HTML |

> MVP 阶段可先用简单的 `<pre>` 标签渲染日志，后续再引入 xterm。

## 7. 开发顺序建议

```
A1 (迁移) + A2 (模型) + A4 (前端类型)
        ↓
A3 (数据库操作) + B1 (加密)
        ↓
B2 (Trait) + B5 (Worktree)
        ↓
B3 (Claude Code) + B4 (Codex) + B6 (健康检查)
        ↓
C1 (配置命令) + C2 (执行命令)
        ↓
C3 (注册)
        ↓
D1 (Store) → D2 (配置界面) → D3 (状态) → D4 (日志)
        ↓
E1 + E2 (测试)
```
