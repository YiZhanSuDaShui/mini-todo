# 任务调度功能 - 调度引擎设计

## 1. 整体架构

调度引擎由以下核心组件构成：

```
┌─────────────────────────────────────────────────┐
│                 TaskScheduler                    │
│  （调度器主控，管理所有调度组件的生命周期）          │
│                                                  │
│  ┌─────────────┐  ┌──────────────┐  ┌─────────┐ │
│  │ PriorityQueue│  │ ConcurrencyMgr│  │ CronMgr │ │
│  │ 优先级队列    │  │ 并发控制       │  │ 定时管理│ │
│  └──────┬──────┘  └──────┬───────┘  └────┬────┘ │
│         │                │                │      │
│  ┌──────┴────────────────┴────────────────┴────┐ │
│  │              TaskExecutor                    │ │
│  │  从队列取任务 → 检查并发 → 调用 AgentManager  │ │
│  └──────────────────────┬──────────────────────┘ │
│                         │                        │
│  ┌──────────────────────┴──────────────────────┐ │
│  │           DependencyResolver                 │ │
│  │  检查依赖是否满足，完成后触发下游              │ │
│  └──────────────────────────────────────────────┘ │
└──────────────────────┬──────────────────────────┘
                       │
          ┌────────────┴────────────────┐
          │     AgentManager (现有)      │
          │  start_background_execution  │
          └─────────────────────────────┘
```

## 2. 优先级队列

### 2.1 优先级计算算法

```rust
fn calculate_priority(subtask: &Subtask, todo: &Todo) -> i64 {
    let mut score: i64 = 0;

    // 1. 基础优先级（来自父 Todo 的优先级）
    score += match todo.quadrant.as_str() {
        "important_urgent" => 30,     // 重要且紧急
        "important_not_urgent" => 20, // 重要不紧急
        "urgent_not_important" => 15, // 紧急不重要
        "not_important_not_urgent" => 10, // 不重要不紧急
        _ => 10,
    } * 3;

    // 2. 等待时间加成（防饥饿）
    // 每等待 10 分钟 +1 分
    let wait_minutes = elapsed_since(subtask.created_at) / 60;
    score += (wait_minutes / 10).min(50) as i64;

    // 3. 截止日期加成（如果父 Todo 有 end_time）
    if let Some(end_time) = &todo.end_time {
        let hours_until_due = hours_until(end_time);
        score += match hours_until_due {
            h if h <= 1 => 50,   // 1 小时内到期
            h if h <= 4 => 30,   // 4 小时内到期
            h if h <= 24 => 15,  // 1 天内到期
            h if h <= 72 => 5,   // 3 天内到期
            _ => 0,
        };
    }

    // 4. 用户手动指定的优先级加成
    score += subtask.priority_score;

    score
}
```

### 2.2 队列数据结构

```rust
use std::collections::BinaryHeap;
use std::cmp::Ordering;

#[derive(Debug, Clone, Eq, PartialEq)]
struct QueuedTask {
    subtask_id: i64,
    todo_id: i64,
    priority: i64,
    enqueued_at: u64,  // 入队时间戳
}

impl Ord for QueuedTask {
    fn cmp(&self, other: &Self) -> Ordering {
        // 优先级高的优先；优先级相同时，先入队的优先
        self.priority
            .cmp(&other.priority)
            .then_with(|| other.enqueued_at.cmp(&self.enqueued_at))
    }
}

impl PartialOrd for QueuedTask {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

struct PriorityQueue {
    heap: BinaryHeap<QueuedTask>,
    max_size: usize,
}

impl PriorityQueue {
    fn new(max_size: usize) -> Self {
        Self { heap: BinaryHeap::new(), max_size }
    }

    fn enqueue(&mut self, task: QueuedTask) -> Result<(), String> {
        if self.heap.len() >= self.max_size {
            return Err("队列已满".to_string());
        }
        self.heap.push(task);
        Ok(())
    }

    fn dequeue(&mut self) -> Option<QueuedTask> {
        self.heap.pop()
    }

    fn peek(&self) -> Option<&QueuedTask> {
        self.heap.peek()
    }

    fn len(&self) -> usize {
        self.heap.len()
    }

    fn remove(&mut self, subtask_id: i64) {
        let items: Vec<_> = self.heap.drain().filter(|t| t.subtask_id != subtask_id).collect();
        self.heap = BinaryHeap::from(items);
    }
}
```

### 2.3 队列持久化

- 应用启动时，从数据库查询 `schedule_status = 'queued'` 的子任务，重建内存队列
- 入队/出队操作同步更新数据库中的 `schedule_status`
- 定期（每 30 秒）重新计算所有排队任务的优先级（等待时间加成变化）

## 3. 并发控制

### 3.1 全局并发管理

```rust
use tokio::sync::Semaphore;
use std::collections::HashMap;
use tokio::sync::RwLock;

struct ConcurrencyManager {
    global_semaphore: Semaphore,           // 全局并发限制
    project_locks: RwLock<HashMap<String, ProjectLock>>,  // 项目级锁
}

struct ProjectLock {
    running_count: usize,
    max_concurrent: usize,  // 默认 1
}

impl ConcurrencyManager {
    fn new(max_global: usize) -> Self {
        Self {
            global_semaphore: Semaphore::new(max_global),
            project_locks: RwLock::new(HashMap::new()),
        }
    }

    /// 尝试获取执行许可
    /// 返回 true 表示可以执行，false 表示需要等待
    async fn try_acquire(&self, project_path: &str) -> bool {
        // 1. 检查全局并发
        if self.global_semaphore.available_permits() == 0 {
            return false;
        }

        // 2. 检查项目级并发
        let locks = self.project_locks.read().await;
        if let Some(lock) = locks.get(project_path) {
            if lock.running_count >= lock.max_concurrent {
                return false;
            }
        }

        true
    }

    /// 获取执行许可（阻塞直到可用）
    async fn acquire(&self, project_path: &str) -> ConcurrencyPermit {
        let global_permit = self.global_semaphore.acquire().await.unwrap();

        // 等待项目级锁
        loop {
            let mut locks = self.project_locks.write().await;
            let lock = locks.entry(project_path.to_string())
                .or_insert(ProjectLock { running_count: 0, max_concurrent: 1 });

            if lock.running_count < lock.max_concurrent {
                lock.running_count += 1;
                break;
            }
            drop(locks);
            tokio::time::sleep(Duration::from_secs(1)).await;
        }

        ConcurrencyPermit {
            project_path: project_path.to_string(),
            _global_permit: global_permit,
        }
    }

    /// 释放执行许可
    async fn release(&self, project_path: &str) {
        let mut locks = self.project_locks.write().await;
        if let Some(lock) = locks.get_mut(project_path) {
            lock.running_count = lock.running_count.saturating_sub(1);
        }
    }
}
```

## 4. 任务依赖链

### 4.1 依赖类型说明

| 类型 | 语义 | 示例 |
|------|------|------|
| finish-to-start（默认） | A 完成后 B 才能开始 | "实现功能" → "编写测试" |
| finish-to-finish | A 完成后 B 才能标记完成 | "代码开发"和"文档编写"可并行，但文档需等代码定稿 |
| start-to-start | A 开始后 B 才能开始 | 并行任务但有启动顺序 |

### 4.2 依赖检查算法

```rust
/// 检查子任务的所有依赖是否满足
fn are_dependencies_met(
    conn: &Connection,
    subtask_id: i64,
) -> Result<bool> {
    let deps = get_dependencies(conn, subtask_id)?;

    for dep in deps {
        let dep_subtask = get_subtask(conn, dep.depends_on_id)?;

        match dep.dependency_type.as_str() {
            "finish-to-start" => {
                // 依赖任务必须已完成或已通过审核
                if !matches!(dep_subtask.schedule_status.as_str(),
                    "completed" | "approved") {
                    return Ok(false);
                }
            }
            "start-to-start" => {
                // 依赖任务必须已开始（至少是 running）
                if !matches!(dep_subtask.schedule_status.as_str(),
                    "running" | "completed" | "approved") {
                    return Ok(false);
                }
            }
            "finish-to-finish" => {
                // 当前任务可以开始，但不能标记完成
                // （此检查在标记完成时进行，不在入队时检查）
                // 所以这里返回 true
            }
            _ => {}
        }
    }
    Ok(true)
}
```

### 4.3 依赖完成触发

```rust
/// 当一个子任务完成时，检查是否有下游任务可以被触发
async fn on_task_completed(
    conn: &Connection,
    completed_subtask_id: i64,
    queue: &mut PriorityQueue,
) -> Result<()> {
    // 查找所有依赖于此任务的下游任务
    let downstream = get_dependents(conn, completed_subtask_id)?;

    for downstream_id in downstream {
        let subtask = get_subtask(conn, downstream_id)?;

        // 只处理 pending 状态的任务
        if subtask.schedule_status != "pending" {
            continue;
        }

        // 检查该任务的所有依赖是否都满足
        if are_dependencies_met(conn, downstream_id)? {
            // 入队
            update_schedule_status(conn, downstream_id, "queued")?;
            let todo = get_todo(conn, subtask.parent_id)?;
            let priority = calculate_priority(&subtask, &todo);
            queue.enqueue(QueuedTask {
                subtask_id: downstream_id,
                todo_id: subtask.parent_id,
                priority,
                enqueued_at: now_ms(),
            })?;
        }
    }
    Ok(())
}
```

## 5. 调度循环

### 5.1 主调度循环

```rust
impl TaskScheduler {
    /// 启动调度器后台任务
    pub fn start(self: Arc<Self>, app: AppHandle) {
        let scheduler = self.clone();

        // 调度循环：每 5 秒检查一次
        tokio::spawn(async move {
            let mut interval = tokio::time::interval(Duration::from_secs(5));

            loop {
                interval.tick().await;
                scheduler.tick(&app).await;
            }
        });

        // 优先级刷新循环：每 30 秒重算
        let scheduler2 = self.clone();
        tokio::spawn(async move {
            let mut interval = tokio::time::interval(Duration::from_secs(30));

            loop {
                interval.tick().await;
                scheduler2.refresh_priorities().await;
            }
        });
    }

    /// 单次调度 tick
    async fn tick(&self, app: &AppHandle) {
        // 1. 检查定时任务
        self.check_cron_tasks(app).await;

        // 2. 检查 pending 任务的依赖是否满足
        self.promote_pending_to_queued(app).await;

        // 3. 从队列取任务执行
        self.dispatch_queued_tasks(app).await;

        // 4. 检查超时任务
        self.check_timeouts(app).await;

        // 5. 处理失败任务的重试
        self.handle_retries(app).await;
    }
}
```

### 5.2 任务分发

```rust
async fn dispatch_queued_tasks(&self, app: &AppHandle) {
    loop {
        // 检查是否有可用的并发额度
        let queue = self.queue.lock().await;
        let task = match queue.peek() {
            Some(t) => t.clone(),
            None => break,
        };
        drop(queue);

        // 获取项目路径
        let project_path = self.get_project_path(app, task.todo_id).await;

        // 检查并发限制
        if !self.concurrency.try_acquire(&project_path).await {
            break; // 没有可用额度，等待下一个 tick
        }

        // 从队列取出任务
        let mut queue = self.queue.lock().await;
        let task = match queue.dequeue() {
            Some(t) => t,
            None => break,
        };
        drop(queue);

        // 异步执行任务
        let scheduler = self.clone();
        let app = app.clone();
        tokio::spawn(async move {
            scheduler.execute_task(&app, task).await;
        });
    }
}
```

### 5.3 任务执行

```rust
async fn execute_task(&self, app: &AppHandle, task: QueuedTask) {
    let db = app.state::<Database>();

    // 1. 更新状态为 running
    db.with_connection(|conn| {
        update_schedule_status(conn, task.subtask_id, "running")
    }).ok();

    // 2. 获取子任务和父 Todo 的信息
    let (subtask, todo) = self.get_task_info(app, task.subtask_id, task.todo_id).await;

    // 3. 构建 prompt
    let prompt = self.build_prompt(&subtask);

    // 4. 获取 agent 配置
    let agent_id = todo.agent_id.unwrap_or(0);
    let project_path = todo.agent_project_path.unwrap_or_default();

    // 5. 生成任务 ID
    let task_id = format!("sched-{}-{}", task.subtask_id, now_ms());

    // 6. 调用现有的 AgentManager 执行
    let agent_manager = app.state::<AgentManager>();
    let result = agent_manager.start_background_execution(
        app.clone(),
        agent_id as i64,
        prompt,
        project_path,
        task_id.clone(),
        Some(task.subtask_id),
    ).await;

    // 7. 等待执行完成（带超时）
    let timeout = Duration::from_secs(subtask.timeout_secs as u64);
    match tokio::time::timeout(timeout, self.wait_for_completion(&task_id)).await {
        Ok(Ok(())) => {
            // 执行成功
            db.with_connection(|conn| {
                update_schedule_status(conn, task.subtask_id, "completed")
            }).ok();

            // 触发下游依赖
            self.on_task_completed(app, task.subtask_id).await;
        }
        Ok(Err(e)) => {
            // 执行失败
            self.handle_task_failure(app, task.subtask_id, &e).await;
        }
        Err(_) => {
            // 超时
            agent_manager.cancel_execution(&task_id).await.ok();
            self.handle_task_failure(app, task.subtask_id, "执行超时，已自动终止").await;
        }
    }

    // 8. 释放并发许可
    self.concurrency.release(&project_path).await;
}
```

## 6. 超时处理

### 6.1 超时机制

```rust
async fn check_timeouts(&self, app: &AppHandle) {
    let agent_manager = app.state::<AgentManager>();
    let states = agent_manager.execution_states.lock().await;

    for (task_id, state) in states.iter() {
        if state.status != "running" {
            continue;
        }

        let elapsed_ms = now_ms() - state.start_time_ms;
        let subtask_id = state.subtask_id.unwrap_or(0);

        // 获取超时配置
        let timeout_secs = self.get_timeout_secs(app, subtask_id).await;
        let timeout_ms = timeout_secs * 1000;

        if elapsed_ms > timeout_ms {
            // 超时 → 取消执行
            let _ = agent_manager.cancel_execution(task_id).await;

            // 更新状态为 failed
            let db = app.state::<Database>();
            db.with_connection(|conn| {
                update_schedule_status(conn, subtask_id, "failed");
                update_schedule_error(conn, subtask_id,
                    &format!("执行超时（{}秒），已自动终止", timeout_secs));
            }).ok();
        }
    }
}
```

## 7. 重试机制

```rust
async fn handle_task_failure(
    &self,
    app: &AppHandle,
    subtask_id: i64,
    error: &str,
) {
    let db = app.state::<Database>();

    let should_retry = db.with_connection(|conn| {
        let subtask = get_subtask(conn, subtask_id)?;

        if subtask.retry_count < subtask.max_retries {
            // 还可以重试
            update_retry_count(conn, subtask_id, subtask.retry_count + 1)?;
            update_schedule_status(conn, subtask_id, "pending")?;
            update_schedule_error(conn, subtask_id,
                &format!("第{}次重试失败: {}", subtask.retry_count + 1, error))?;
            Ok(true)
        } else {
            // 超过重试次数，标记为最终失败
            update_schedule_status(conn, subtask_id, "failed")?;
            update_schedule_error(conn, subtask_id,
                &format!("已达最大重试次数({}): {}", subtask.max_retries, error))?;
            Ok(false)
        }
    }).unwrap_or(false);

    if should_retry {
        // 延迟重试：指数退避
        let delay = Duration::from_secs(30 * 2u64.pow(retry_count as u32));
        tokio::time::sleep(delay).await;
        // 重新进入 pending 状态后，下一次 tick 会重新检查依赖并入队
    }
}
```

## 8. 并发冲突解决策略

### 8.1 策略选择

```rust
enum ConflictStrategy {
    MutexLock,      // 互斥锁（默认）：同一项目串行执行
    BranchIsolation, // 分支隔离：每个任务独立分支
    WorktreeIsolation, // Worktree 隔离：独立工作目录
}
```

### 8.2 互斥锁策略（默认推荐）

最简单且最安全的策略。同一项目同时只有一个 Agent 任务运行，其他任务在队列中等待。

- 优点：零冲突风险，实现简单
- 缺点：并行度低
- 适用：大多数场景

### 8.3 分支隔离策略

```rust
async fn execute_with_branch_isolation(
    &self,
    project_path: &str,
    subtask_id: i64,
    prompt: &str,
) -> Result<(), String> {
    let branch_name = format!("agent/task-{}", subtask_id);

    // 1. 创建新分支
    run_git(project_path, &["checkout", "-b", &branch_name])?;

    // 2. 执行 Agent 任务
    let result = self.run_agent(project_path, prompt).await;

    // 3. 尝试合并回主分支
    let current_branch = run_git(project_path, &["rev-parse", "--abbrev-ref", "HEAD"])?;
    run_git(project_path, &["checkout", &current_branch])?;

    match run_git(project_path, &["merge", &branch_name, "--no-edit"]) {
        Ok(_) => {
            // 合并成功，删除临时分支
            run_git(project_path, &["branch", "-d", &branch_name])?;
            Ok(())
        }
        Err(_) => {
            // 合并冲突
            run_git(project_path, &["merge", "--abort"])?;

            // 尝试 rebase
            run_git(project_path, &["checkout", &branch_name])?;
            match run_git(project_path, &["rebase", &current_branch]) {
                Ok(_) => {
                    run_git(project_path, &["checkout", &current_branch])?;
                    run_git(project_path, &["merge", &branch_name])?;
                    run_git(project_path, &["branch", "-d", &branch_name])?;
                    Ok(())
                }
                Err(_) => {
                    run_git(project_path, &["rebase", "--abort"])?;
                    Err("Git 合并冲突，需要手动解决".to_string())
                }
            }
        }
    }
}
```

### 8.4 策略配置

- 默认使用互斥锁策略
- 可在 Todo 级别配置 `conflict_strategy` 字段（未来扩展）
- 分支隔离和 Worktree 隔离策略为进阶功能
