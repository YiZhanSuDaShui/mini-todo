# 智能化能力 - 前端实现方案

## 1. 新增组件清单

| 组件 | 位置 | 说明 |
|------|------|------|
| `TaskSplitDialog.vue` | `src/components/` | 任务拆分主对话框 |
| 无需新增 Store | - | 逻辑集中在组件内部，通过 `invoke` 直接调用后端 |

## 2. 交互流程

```
EditorView（大任务编辑页面）
  ├── [现有] Agent 配置按钮
  └── [新增] "智能拆分" 按钮 ← 仅在已配置 Agent + 项目路径时显示
        ↓ 点击
      TaskSplitDialog 打开
        ├── 步骤 1：输入需求
        │     ├── 需求输入框（textarea）
        │     └── [拆分] 按钮
        │           ↓ 调用 split_task 命令
        ├── 步骤 2：预览结果
        │     ├── 拆分结果列表（可编辑）
        │     ├── 依赖关系可视化
        │     ├── [重新拆分] 按钮
        │     └── [创建子任务] 按钮
        │           ↓ 调用 apply_split_result 命令
        └── 步骤 3：完成
              └── 显示创建成功的子任务列表
                    ↓ 关闭对话框
              EditorView 刷新子任务列表
```

export const AGENT_LABELS: Record<string, string> = {

}

## 5. EditorView 集成

### 5.1 触发入口位置

在 EditorView 的操作栏（现有 Agent 配置按钮旁边）添加"智能拆分"按钮：

```html
<el-button
  v-if="isEdit && todo?.agentProjectPath"
  type="primary"
  plain
  size="small"
  @click="showSplitDialog = true"
>
  <el-icon><MagicStick /></el-icon>
  智能拆分
</el-button>
```

显示条件：
- 处于编辑模式（`isEdit`）
- 已配置项目路径（`todo?.agentProjectPath`）

### 5.2 对话框挂载

```html
<TaskSplitDialog
  v-if="showSplitDialog"
  :todo-id="todoId"
  :project-path="todo.agentProjectPath"
  @completed="onSplitCompleted"
  @close="showSplitDialog = false"
/>
```

`onSplitCompleted` 回调中刷新子任务列表。

## 6. 交互细节

### 6.1 任务行操作

- **点击行**：展开/折叠任务详情
- **拖拽排序**：调整任务顺序（使用 vuedraggable）
- **删除按钮**：删除单个任务，自动更新依赖索引
- **编辑字段**：直接修改标题、描述、Prompt、复杂度、Agent、依赖

### 6.2 依赖关系编辑

- 依赖选择使用 `el-select` 多选模式
- 选项为当前任务之前的所有任务（不能依赖自己或后面的任务）
- 删除任务时自动清理相关依赖

### 6.3 错误处理

- 拆分超时：显示"AI 响应超时，请重试"
- 解析失败：显示原始 AI 输出，让用户手动判断
- 创建失败：显示具体错误，已创建的子任务保留

## 7. 样式设计

保持与现有 UI 风格一致：

- 对话框宽度：`width="680px"`，`top="5vh"`
- 内容区域：`max-height: calc(90vh - 160px); overflow-y: auto`
- 任务表格：使用 `el-table` + 自定义展开行
- 复杂度标签：使用 `el-tag`，颜色对应 `COMPLEXITY_MAP`
- 加载动画：使用 `el-loading` 全屏遮罩
