### 3.2 步骤编辑交互

每个步骤包含：
- **类型选择**：`el-select` 下拉框选择「执行子任务」或「执行提示词」
- **内容配置**：
  - 子任务类型：`el-select` 选择已有子任务（排除已被其他步骤引用的）
  - 提示词类型：`el-input` 文本输入框（支持多行）
- **排序按钮**：上移 / 下移
- **删除按钮**：移除步骤

### 3.3 工作流表单数据

```typescript
const workflowSteps = ref<WorkflowStepInput[]>([])
const workflowEnabled = ref(false)

// 加载已有步骤
async function loadWorkflowSteps() {
  if (!todoId.value) return
  const steps = await invoke<WorkflowStep[]>('get_workflow_steps', {
    todoId: todoId.value
  })
  workflowSteps.value = steps.map(s => ({
    stepType: s.stepType,
    subtaskId: s.subtaskId,
    promptText: s.promptText,
  }))
}

// 添加步骤
function addStep() {
  workflowSteps.value.push({
    stepType: 'subtask',
    subtaskId: undefined,
    promptText: undefined,
  })
}

// 删除步骤
function removeStep(index: number) {
  workflowSteps.value.splice(index, 1)
}

// 上移/下移
function moveStep(index: number, direction: 'up' | 'down') {
  const target = direction === 'up' ? index - 1 : index + 1
  if (target < 0 || target >= workflowSteps.value.length) return
  const temp = workflowSteps.value[index]
  workflowSteps.value[index] = workflowSteps.value[target]
  workflowSteps.value[target] = temp
}

// 保存工作流
async function saveWorkflow() {
  await invoke('set_workflow_steps', {
    todoId: todoId.value,
    steps: workflowSteps.value,
  })
  ElMessage.success('工作流已保存')
}

// 启动工作流
async function startWorkflow() {
  try {
    await invoke('start_workflow', { todoId: todoId.value })
    ElMessage.success('工作流已启动')
    loadTodo()
  } catch (e) {
    ElMessage.error('启动失败: ' + String(e))
  }
}
```

### 3.4 子任务选择过滤

function availableSubtasks(currentIndex: number) {
  const usedIds = workflowSteps.value
    .filter((s, i) => i !== currentIndex && s.stepType === 'subtask' && s.subtaskId)

### 5.2 执行模式（工作流运行中）

```html
<el-button @click="pauseWorkflow">暂停</el-button>
<el-button @click="skipStep">跳过当前步骤</el-button>
<el-button type="danger" @click="resetWorkflow">重置</el-button>
```

### 5.3 控制函数

```typescript
async function pauseWorkflow() {
  await invoke('pause_workflow', { todoId: todoId.value })
  ElMessage.info('工作流已暂停')
  loadTodo()
}

async function skipStep() {
  await invoke('skip_workflow_step', { todoId: todoId.value })
  ElMessage.info('已跳过当前步骤')
  loadTodo()
}

async function resetWorkflow() {
  try {
    await ElMessageBox.confirm('确定要重置工作流吗？所有步骤将回到待执行状态。', '确认重置')
    await invoke('reset_workflow', { todoId: todoId.value })
    ElMessage.success('工作流已重置')
    loadTodo()
  } catch {}
}
```

## 7. 数据流总结

```
配置阶段：
  EditorView → 步骤编辑器 → 添加/排序/删除步骤
    ↓ 保存
  invoke('set_workflow_steps') → workflow_steps 表

启动阶段：
  invoke('start_workflow')
    → workflow_current_step = 0
    → 执行第一步

执行阶段（子任务步骤）：
  workflow.rs → 将子任务状态设为 pending
    → 调度器拾取 → Agent 执行
    → engine.rs 完成回调
    → workflow::advance_workflow()
    → 执行下一步

执行阶段（提示词步骤）：
  workflow.rs → AgentManager.start_background_execution()
    → 轮询等待完成
    → advance_workflow()
    → 执行下一步

用户控制：
  暂停 → invoke('pause_workflow') → 停止自动推进
  跳过 → invoke('skip_workflow_step') → 标记 skipped → 推进下一步
  重置 → invoke('reset_workflow') → 所有步骤 pending → current_step = -1
```
