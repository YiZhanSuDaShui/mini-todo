## 灵感梳理：Mini-Todo -> AI 任务调度 Agents 系统

### 1. 核心概念定位

将 Mini-Todo 从"个人待办管理"升级为代码任务的调度中心。用户在 Todo 中描述需求，系统自动或手动将任务派发给 AI Agent（Claude Code / Codex），Agent 执行后返回结果，用户审核确认。

类比：你就像是一个项目经理，Mini-Todo 就是你的"JIRA"，而 Claude Code 和 Codex 是你的"开发团队"。

------

### 2. Agent 集成层

| 灵感           | 描述                                                         |
| :------------- | :----------------------------------------------------------- |
| 多 Agent 配置  | 在设置页面配置多个 Agent 连接（Claude Code CLI、Codex CLI），包括路径、API Key、默认参数等 |
| Agent 能力画像 | 每个 Agent 标注其擅长领域（如 Claude Code 擅长复杂架构、Codex 擅长快速小改动），用于智能分配 |
| Agent 健康检查 | 检测 Agent CLI 是否可用、API Key 是否有效、余额是否充足      |
| 执行环境隔离   | 每个 Agent 任务运行在指定的工作目录/Git 分支中，避免冲突     |

### 3. 任务调度功能

| 灵感            | 描述                                                         |
| :-------------- | :----------------------------------------------------------- |
| 任务类型扩展    | 在现有待办基础上新增"Agent 任务"类型，包含：目标项目路径、提示词(Prompt)、指定 Agent、执行策略 |
| Prompt 模板系统 | 预设常用模板（Bug 修复、功能开发、代码重构、代码审查、单元测试编写等），用户填入变量即可生成标准化 Prompt |
| 任务依赖链      | 支持设置任务前置依赖（任务 B 等待任务 A 完成后自动启动），形成 DAG 调度图 |
| 定时/条件触发   | 支持 Cron 定时执行、或基于 Git 事件（如新 commit、新 issue）自动触发 Agent 任务 |
| 并发控制        | 设置最大并行 Agent 数量，避免资源争抢                        |
| 优先级队列      | 高优先级任务自动插队执行                                     |

### 4. 执行与监控

| 灵感         | 描述                                                         |
| :----------- | :----------------------------------------------------------- |
| 实时执行日志 | Agent 执行过程中实时流式展示输出日志（类似终端输出）         |
| 执行状态追踪 | 待调度 -> 排队中 -> 执行中 -> 等待审核 -> 已完成/已拒绝      |
| 超时与重试   | 设置任务超时时间，支持自动重试或手动重试                     |
| 成本追踪     | 记录每次 Agent 调用消耗的 Token 数和费用                     |
| 执行历史     | 完整的历史记录，包含 Prompt、Agent 输出、耗时、成本、结果状态 |

### 5. 结果管理与审核

| 灵感          | 描述                                                   |
| :------------ | :----------------------------------------------------- |
| Diff 预览     | Agent 完成后展示代码变更的 Diff 视图（类似 GitHub PR） |
| 一键审批/拒绝 | 审核通过后自动 commit，拒绝则回滚或重新派发            |
| Git 集成      | Agent 在独立分支工作，完成后自动创建 PR，用户 Merge    |
| 结果评分      | 对 Agent 执行结果打分，积累数据用于后续智能分配        |
| 反馈循环      | 拒绝时可附加修改意见，Agent 基于反馈重新执行           |

### 6. 智能化能力

| 灵感            | 描述                                                         |
| :-------------- | :----------------------------------------------------------- |
| 任务智能拆分    | 输入一个大需求，AI 自动拆分为多个可执行的子任务              |
| 上下文注入      | 自动分析目标项目的技术栈、代码结构，注入到 Agent 的 Prompt 中 |
| 智能 Agent 分配 | 根据任务类型和历史表现，自动推荐最合适的 Agent               |
| 自然语言调度    | 支持用自然语言描述调度策略（如"每天早上 9 点检查所有项目的 TODO 注释"） |

### 7. 项目管理增强

| 灵感           | 描述                                                         |
| :------------- | :----------------------------------------------------------- |
| 多项目支持     | 管理多个代码项目，每个项目有独立的 Agent 任务队列            |
| 项目上下文配置 | 每个项目可配置：仓库路径、技术栈描述、编码规范、自定义指令文件（类似 CLAUDE.md） |
| 看板视图       | 新增看板模式，按状态列（待调度/执行中/待审核/已完成）展示 Agent 任务 |
| 统计仪表盘     | 任务完成率、Agent 使用统计、成本趋势图、效率分析             |

### 8. 工作流引擎（进阶）

| 灵感         | 描述                                                         |
| :----------- | :----------------------------------------------------------- |
| 可视化流水线 | 拖拽式编排多步骤 Agent 工作流（如：分析需求 -> 编写代码 -> 写测试 -> Code Review） |
| 条件分支     | 工作流中支持条件判断（如测试通过则继续，失败则通知人工）     |
| 人机协作节点 | 工作流中插入人工审核节点，等待用户确认后继续                 |
| 工作流模板   | 预设常见工作流（Feature 开发、Bug 修复、代码迁移等）         |

### 9. 技术实现方向

| 方面       | 建议                                                         |
| :--------- | :----------------------------------------------------------- |
| Agent 通信 | 通过 Rust 后端调用 Claude Code CLI / Codex CLI（子进程），解析 stdout 输出 |
| 任务队列   | SQLite + 定时轮询 或 内存队列，管理 Agent 任务状态           |
| 实时日志   | 使用 Tauri 的事件系统（`emit`/`listen`）从 Rust 后端推送 Agent 输出到前端 |
| Diff 渲染  | 前端集成 `diff2html` 或 `monaco-editor` 的 diff 视图         |
| 工作流存储 | JSON 格式存储工作流定义，SQLite 存储执行实例                 |

------

这些灵感涵盖了从基础 Agent 对接到高级工作流编排的完整路径。你可以根据优先级分阶段实现，建议的路线图是：

1. MVP：Agent 配置 + 单任务派发 + 执行日志 + 结果查看
2. V2：Prompt 模板 + 任务依赖 + Git 集成 + Diff 审核
3. V3：多项目 + 智能分配 + 统计仪表盘
4. V4：可视化工作流引擎 + 条件触发 + 反馈循环
